A Media Type for XML Patch Operations
draft-wilde-xml-patch-00

Abstract

The XML Patch media type "application/xml-patch" defines an XML document structure for expressing a sequence of operations to apply to an XML document. The XML Patch document format is defined in RFC 5261, this specification simply is a media type registration, so that XML Patch documents can be labeled with a media type, for example in HTTP conversations.

1.  Introduction

The Extensible Markup Language (XML) [1] (Murata, M., St. Laurent, S., and D. Kohn, “XML Media Types,” January 2001.) is a common format for the exchange and storage of structured data. HTTP PATCH [6] (Dusseault, L. and J. Snell, “PATCH Method for HTTP,” March 2010.) extends HTTP [7] (Fielding, R., Gettys, J., Mogul, J., Frystyk, H., Masinter, L., Leach, P., and T. Berners-Lee, “Hypertext Transfer Protocol -- HTTP/1.1,” June 1999.) with a method to perform partial modifications to resources.

The XML Patch media type "application/xml-patch" is an XML document structure for expressing a sequence of operations to apply to a target XML document, suitable for use with the HTTP PATCH method. It also defines an error document structure that allows servers to communicate problems within patch documents.

The formats for the patch and error documents are based on the XML Parch Framework defined in RFC 5261 [2] (Urpalainen, J., “An Extensible Markup Language (XML) Patch Operations Framework Utilizing XML Path Language (XPath) Selectors,” September 2008.).

2.  IANA Considerations

The Internet media type [3] (Freed, N. and J. Klensin, “Media Type Specifications and Registration Procedures,” December 2005.) for an XML Patch Document is application/xml-patch.

Type name: application

Subtype name: xml-patch

Required parameters: none

Optional parameters: none

Encoding considerations: Same as encoding considerations of application/xml as specified in [1] (Murata, M., St. Laurent, S., and D. Kohn, “XML Media Types,” January 2001.).

Security considerations: This media type has all of the security considerations described in [1] (Murata, M., St. Laurent, S., and D. Kohn, “XML Media Types,” January 2001.), plus those listed in Section 3 (Security Considerations).

Interoperability considerations: N/A

Published specification: [[ This document ]]

Applications that use this media type: Applications that manipulate XML documents.

Additional information:

Magic number(s): N/A

File extension(s): XML data should use ".xml" as the file extension.

Macintosh file type code(s): TEXT

Person & email address to contact for further information: Erik Wilde <erik.wilde@emc.com>

Intended usage: COMMON

Restrictions on usage: none

Author: Erik Wilde <erik.wilde@emc.com>

Change controller: IETF
